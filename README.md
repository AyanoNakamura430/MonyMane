# 家計簿管理アプリ

ブラウザ上で収入・固定費・変動費・カテゴリ予算を管理するための家計簿アプリです。月額/年額の集計、変動費予算の進捗確認、データのバックアップ/復元に対応しています。

## 主な機能

- ホーム画面での家計集計
  - 今月/今年あといくら使えるか
  - 貯金目標を差し引いた自由予算
  - 収入、固定費、変動費の合計
  - 年間収入/年間固定費の見積
  - 変動費予算の使用率とプログレスバー
  - カテゴリ別の予算進捗
- 登録/編集画面
  - 収入
  - 固定費
  - 変動費実績
  - カテゴリ
  - 変動費カテゴリ予算
  - 年間収入
  - 年間固定費
  - 貯金目標
- 設定画面
  - JSONエクスポート
  - JSONインポート
  - 家計簿関連データの一括削除

## 使い方

1. ホーム画面で対象月を確認します。
2. 「登録・編集」画面で、収入や支出を登録します。
3. 変動費を分類したい場合は、先に「カテゴリ」を登録します。
4. 「変動費カテゴリ予算」でカテゴリごとの予算を登録します。
5. 変動費実績の登録時にカテゴリを選択すると、カテゴリ別の進捗に反映されます。未選択の場合は「未分類」として扱われます。
6. ホーム画面で、予算合計、実績合計、残り合計、使用率、カテゴリ別進捗を確認します。
7. データを残したい場合は、設定画面からJSONエクスポートを行います。

## データ保存方式

このアプリのデータは、ブラウザの`localStorage`に保存されます。サーバーや外部サービスには保存されません。

主な保存対象は以下です。

- 収入
- 固定費
- 変動費実績
- カテゴリ
- 変動費カテゴリ予算
- 年間収入
- 年間固定費
- 貯金目標

## プライバシー

- 入力した家計簿データは、利用中のブラウザ内に保存されます。
- 現時点では、入力データをサーバーへ送信したり、外部サービスへ共有したりする処理はありません。
- 端末やブラウザを変えた場合、データは自動共有されません。
- ブラウザのデータ削除、シークレットモードの終了、端末故障などにより、保存済みデータが失われる可能性があります。
- バックアップが必要な場合は、設定画面のJSONエクスポート機能を使用してください。
- 復元する場合は、設定画面のJSONインポート機能でエクスポート済みファイルを選択してください。インポート時は現在の家計簿データが上書きされます。

## 開発環境での起動

依存関係をインストールします。

```bash
npm install
```

開発サーバーを起動します。

```bash
npm run dev
```

本番ビルドを確認します。

```bash
npm run build
```

ビルド結果をローカルで確認します。

```bash
npm run preview
```

## デプロイ

Vercelで公開する場合は、Viteアプリとしてそのままデプロイできます。

推奨設定は以下です。

- Framework Preset: `Vite`
- Install Command: `npm install`
- Build Command: `npm run build`
- Output Directory: `dist`
- Environment Variables: 不要

このアプリは画面切り替えをアプリ内部の状態で管理しており、現時点ではURLベースのルーティングを使用していません。そのため、Vercel用の`vercel.json`は必須ではありません。

データはブラウザの`localStorage`に保存されるため、公開環境でもサーバー側のデータベースやAPI設定は不要です。

## 注意事項

- このアプリはブラウザ内保存を前提としています。重要なデータは定期的にJSONエクスポートしてください。
- 一括削除を実行すると、家計簿アプリ関連データのみ削除されます。ただし、削除後はバックアップがない限り復元できません。
- インポートするJSONは、このアプリでエクスポートしたバックアップファイルを使用してください。
- 複数端末で同じデータを使いたい場合は、エクスポートしたJSONを別端末でインポートしてください。
