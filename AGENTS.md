# AGENTS.md

## プロジェクト概要

MonyManeは、収入・固定費・変動費・月間予算・年間見積・貯金目標をブラウザ上で管理する家計簿アプリです。v1.0.0をリリース済みです。データはサーバーへ保存せず、利用中のブラウザの`localStorage`へ保存します。

## 現行実装の参照先

現行仕様を調査・変更するときは、次のファイルを正として扱ってください。

- エントリーポイント: `src/main.tsx` → `src/app/App.tsx`
- データ型: `src/app/types.ts`
- localStorage、データ移行、集計処理: `src/app/useBudgetData.ts`
- ビルド・開発コマンド: `package.json`
- v1.0.0仕様: `docs/SPEC.md`
- 今後の作業候補: `docs/TASKS.md`
- リリース確認項目: `RELEASE_CHECKLIST.md`

## 改修対象と未参照コード

- 通常の改修対象は、現行エントリーポイントから参照されるコードです。
- `src/app/components`および`src/imports`には、現行エントリーポイントから参照されていないコードがあります。
- 未参照コードは削除候補ですが、明示的な依頼がない限り、調査・変更・削除しないでください。
- 未参照コードを現行仕様の根拠として扱わないでください。

## 変更の基本ルール

- 依頼範囲外の機能追加、最適化、依存関係更新、ファイル整理を行わないでください。
- 大規模なリファクタリングや`App.tsx`の分割を無断で行わないでください。
- 変更前に対象コード、`docs/SPEC.md`、関連ドキュメントを確認してください。
- 実装とドキュメントが矛盾する場合は、推測で統一せず、差異を報告してください。
- 仕様を判断できない場合は、実装前に確認してください。

## データ互換性

次のデータとの互換性を維持してください。

- v1.0.0で保存されたlocalStorageデータ
- backup version 1
- backup version 2

`src/app/useBudgetData.ts`のlocalStorageキー、保存スキーマ、データバージョン、移行処理を無断で変更・削除してはいけません。バックアップ形式を変更するときは、`src/app/App.tsx`のエクスポート・インポート処理も同時に確認してください。

データ構造を変更する場合は、次をすべて満たす必要があります。

1. 新しいデータバージョンと移行元・移行先を定義する。
2. 既存データを失わない移行処理を実装する。
3. 移行処理を複数回実行してもデータを重複・破損させない。
4. localStorageが空、不正、部分的、または利用不可の場合を考慮する。
5. backup version 1および2の復元可否を確認する。
6. `docs/SPEC.md`と必要なリリース確認項目を更新する。

## UIの基本方針

- 最新版のChrome、Edge、Safariを対象とします。
- 最小画面幅は320pxです。
- 既存のレイアウト、色、タイポグラフィ、操作方法を、依頼なしに変更しないでください。
- 主要操作はモバイル幅でも利用できる状態を維持してください。
- エラー、成功、予算超過、危険操作は、色だけに依存せず文言でも伝えてください。
- アイコンだけのボタンには識別可能なラベルを維持してください。

## 金額・日付・カテゴリ

- 通貨は日本円として表示します。
- 通常の収入・支出・年間見積・貯金目標は1円以上です。
- 月間の固定費予算・変動費予算は0円以上です。
- 金額入力は数字以外を除去し、保存時は`number`として扱います。
- 収入・固定費・変動費の`month`は日付文字列として扱い、月次集計では先頭7文字の`YYYY-MM`を使用します。
- 月間予算の`yearMonth`は`YYYY-MM`です。
- カテゴリは「グループカテゴリ → カテゴリ」の2階層です。
- 変動費ではグループとカテゴリを省略でき、省略した実績は未分類として集計します。
- グループを選択した変動費では、その配下のカテゴリを選択する必要があります。

## 削除制約

- 配下カテゴリがあるグループカテゴリは削除できません。
- 変動費実績から参照されているグループカテゴリは削除できません。
- 変動費実績から参照されているカテゴリは削除できません。
- 全削除は既知の家計簿用localStorageキーだけを対象とし、他のキーを削除してはいけません。
- 破壊的操作の確認手順を無断で省略しないでください。

## Git確認

- 作業前に`git status --short`を確認し、既存の変更を把握してください。
- ユーザーの既存変更を上書き、破棄、巻き戻ししないでください。
- 作業後に`git diff --check`、`git diff`、`git status --short`を確認してください。
- 明示的な依頼がない限り、コミット、タグ作成、pushを行わないでください。

## 検証コマンド

現在`package.json`で利用可能なコマンドは次のとおりです。

```bash
npm run dev
npm run build
npm run preview
npm run typecheck
```

- パッケージマネージャーはnpmです。
- `npm run typecheck`は`src/main.tsx`を起点に、現行アプリから参照されるコードだけを検査します。
- `vite.config.ts`、`src/app/components`および`src/imports`の未参照コードは、現在のtypecheck対象外です。
- lint、format、unit test、component test、E2E testは現時点で未整備です。
- 存在しないコマンドを実行可能として案内しないでください。
- Node.jsの正式な対応バージョンは未確定です。ローカルで動作したバージョンを、そのまま対応範囲として確定しないでください。
