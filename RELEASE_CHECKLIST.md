# リリースチェックリスト

公開前に以下を確認してください。

## 1. ビルド

- [ ] `npm install`が成功する
- [ ] `npm run build`が成功する
- [ ] `npm run preview`で画面が表示される

## 2. 基本CRUD

- [ ] 収入を登録できる
- [ ] 収入を編集できる
- [ ] 収入を削除できる
- [ ] 固定費を登録できる
- [ ] 固定費を編集できる
- [ ] 固定費を削除できる
- [ ] 変動費実績を登録できる
- [ ] 変動費実績を編集できる
- [ ] 変動費実績を削除できる
- [ ] カテゴリを登録できる
- [ ] カテゴリを編集できる
- [ ] 未使用カテゴリを削除できる
- [ ] 使用中または予算ありカテゴリを削除できない
- [ ] 変動費カテゴリ予算を登録できる
- [ ] 変動費カテゴリ予算を編集できる
- [ ] 変動費カテゴリ予算を削除できる
- [ ] 年間収入を登録できる
- [ ] 年間収入を編集できる
- [ ] 年間収入を削除できる
- [ ] 年間固定費を登録できる
- [ ] 年間固定費を編集できる
- [ ] 年間固定費を削除できる
- [ ] 貯金目標を保存できる

## 3. グラフ/集計

- [ ] ホーム画面に予算合計が表示される
- [ ] ホーム画面に実績合計が表示される
- [ ] ホーム画面に残り合計が表示される
- [ ] ホーム画面に使用率が整数%で表示される
- [ ] 変動費予算のプログレスバーが表示される
- [ ] カテゴリ別進捗が表示される
- [ ] 予算0円でも表示が崩れない
- [ ] 実績0円でも表示が崩れない
- [ ] 使用率100%でも表示が崩れない
- [ ] 使用率100%超過時に「予算オーバー」が表示される
- [ ] 月額/年額モード切り替えで値が切り替わる

## 4. localStorage

- [ ] localStorageが空でも起動できる
- [ ] 初回起動状態でエラーが出ない
- [ ] 登録後にリロードしてもデータが残る
- [ ] 旧`budget-app-variable-categories`データがある場合、新形式へ補完される
- [ ] 家計簿関連以外のlocalStorageキーを削除しない

## 5. インポート/エクスポート

- [ ] JSONエクスポートができる
- [ ] エクスポートファイル名が`budget-app-backup-YYYYMMDD-HHmmss.json`形式である
- [ ] エクスポートJSONに収入が含まれる
- [ ] エクスポートJSONに固定費が含まれる
- [ ] エクスポートJSONに変動費実績が含まれる
- [ ] エクスポートJSONにカテゴリが含まれる
- [ ] エクスポートJSONに変動費カテゴリ予算が含まれる
- [ ] エクスポートJSONに年間収入が含まれる
- [ ] エクスポートJSONに年間固定費が含まれる
- [ ] エクスポートJSONに貯金目標が含まれる
- [ ] 正常なバックアップJSONをインポートできる
- [ ] インポート前に上書き確認が表示される
- [ ] 不正JSONをインポートするとエラー表示される
- [ ] アプリ用でないJSONをインポートするとエラー表示される
- [ ] インポート後に画面表示が更新される

## 6. 一括削除

- [ ] 一括削除エリアが設定画面にある
- [ ] 「削除」と入力しないと一括削除できない
- [ ] 削除前に確認ダイアログが表示される
- [ ] 一括削除後、家計簿関連データのみ削除される
- [ ] 一括削除後、初回起動相当の空状態になる
